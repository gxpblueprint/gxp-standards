# Contributing to GxP Standards

We welcome contributions from validation engineers, QA professionals, and industry experts.

## How to Contribute
1. **Fork** this repository.
2. **Branch** off `main` (e.g., `feature/update-gdp-sop`).
3. **Edit** the Markdown files.
4. **Submit a Pull Request (PR)**.

## Style Guide
* Use standard Markdown.
* Use "Admonitions" (e.g., `:::info`) for critical regulatory notes.
* Keep language concise, active, and audit-ready.
